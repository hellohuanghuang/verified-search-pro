# 输出模板规范

## 默认输出格式

- **格式**: Markdown (`.md`)
- **编码**: UTF-8
- **标题层级**: 严格使用 `#` / `##` / `###` / `####`, 禁止跳级
- **语言**: 中文为主, 专业术语保留英文
- **语气**: 专业、客观、有依据

## 2.0 结构化输出格式

当报告需要被另一个 agent、benchmark harness 或审计脚本消费时，使用:

```bash
python3 scripts/search_engine.py "query" --verify --output claims-json
```

`claims-json` 必须保留:

- `claims`: 最小可验证 claim、置信度、限制项
- `evidence`: URL、标题、摘要、来源引擎、访问时间
- `source_reliability`: 来源可靠性分级
- `information_credibility`: 内容可信度分级
- `freshness`: 发布日期和时效状态
- `limitations`: 信息不足、未验证、来源未知等限制

```markdown
> **查询**: [原始查询]
> **日期**: [日期]
> **引擎**: [使用的引擎]
> **结果数**: [去重后数量]

---

## 核心结论 (要点锚定)

1. **[结论1]** (置信度: [A/B/C/D/E])
   - 来源: [来源列表]
   - 依据: [简要说明]

2. **[结论2]** (置信度: [A/B/C/D/E])
   ...

## 详细发现

### [模块1]

- **[发现1]** (置信度: [等级])
  - 来源: [URL]
  - 摘要: [内容摘要]

- **[发现2]** ...

### [模块2]
...

- **[问题1]**: [说明信息不足的原因] (置信度: E)
- **[问题2]**: [说明矛盾情况] (置信度: D)

## 附录: 原始结果清单

[按需附上完整结果列表]
```

## 交付方式

| 目标 | 处理方式 | 状态 |
|------|---------|------|
| **飞书文档** | `feishu_create_doc` / `feishu_update_doc` | ✅ 支持 |
| **本地文件** | 直接写入 workspace | ✅ 支持 |
| **Obsidian** | 兼容 Wikilink | ✅ 支持 |

## 质量控制检查清单

交付前必须检查:

- [ ] 每个核心结论都有置信度标注
- [ ] 信息不足项明确标注, 不强行填充
- [ ] 矛盾信息标注争议, 不自行判断
- [ ] 来源分布表完整
- [ ] 无 AI 腔调, 表述客观
- [ ] 专业术语已解释或标注
- [ ] 格式符合 Markdown 规范
