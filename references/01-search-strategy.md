# 搜索策略与引擎选择指南

## 何时使用哪个引擎？

| 场景 | 首选引擎 | 辅助引擎 | 原因 |
|------|---------|---------|------|
| **中文事实核查** | 百度 + 必应 | Tavily | 中文网页覆盖更广 |
| **英文/国际信息** | Tavily + 必应国际 | 必应 | Tavily 英文质量高 |
| **科技/行业动态** | Tavily | 百度 | Tavily 深度搜索更好 |
| **微信公众号内容** | 搜狗微信 | 百度 | 搜狗微信搜索专精 |
| **政策/政府信息** | 百度 | 必应 | 百度对 gov.cn 收录好 |
| **通用快速查询** | Tavily | 必应 | Tavily 速度+质量平衡 |

## 查询拆分策略

复杂查询拆分为多个子查询：

```
原查询："小鹏汽车 VLA2.0 激光雷达 技术路线"
子查询1："小鹏 VLA2.0 技术架构"
子查询2："小鹏 纯视觉 激光雷达 取舍"
子查询3："VLA2.0 vs 激光雷达 智驾方案对比"
```

## 预算控制

| 预算 | 结果数 | 适用场景 |
|------|--------|---------|
| minimal | 5 | 快速事实核查 |
| balanced | 10 | 标准调研 |
| comprehensive | 20 | 深度研究 |

## 引擎健康监控

每个引擎有独立的失败计数器：
- 连续失败 3 次 → 熔断 5 分钟
- 成功后重置失败计数
- 熔断期间跳过该引擎
