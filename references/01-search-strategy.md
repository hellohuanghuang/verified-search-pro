# 搜索策略与引擎选择指南

## 何时使用哪个引擎？

| 场景 | 首选引擎 | 辅助引擎 | 原因 |
|------|---------|---------|------|
| **中文事实核查** | 百度 + 必应 | Tavily | 中文网页覆盖更广 |
| **英文/国际信息** | Tavily + 必应国际 | 必应 | Tavily 英文质量高 |
| **科技/行业动态** | Tavily | 百度 | Tavily 深度搜索更好 |
| **微信公众号内容** | 搜狗微信 | 百度 | 搜狗微信搜索专精 |
| **政策/政府信息** | 百度 | 必应 | 百度对 gov.cn 收录好 |
| **通用快速查询** | Tavily | 必应 | Tavily 速度+质量平衡 |
| **观点/争议/评价** | 必应 + 百度 | Tavily | 先覆盖多元材料，再在输出端清晰标注 |

## 2.0 模式选择

| 模式 | 适用任务 | 输出重点 |
|------|----------|----------|
| `fact` | 事实核查、确认某事是否成立 | 可信结论、证据、置信度、限制 |
| `perspective` | 主观观点、评价、争议、批评 | 观点地图、代表来源、争议不确定、不可直接当事实标注 |
| `research` | 深度调研、政策/机构/项目研究 | 可信结论 + 观点/误区/时间演进的综合资料包 |
| `auto` | 用户未指定时 | 根据查询词保守判断 |

## Google 策略

Google 暂不进入默认能力。它可以作为未来的 `google_cse` 可选适配，但需要单独的 API key、Programmable Search Engine 配置、可用网络/代理和配额管理。2.0 默认优先保证公开安装后的稳定降级，而不是追求最多搜索源。

## 查询拆分策略

复杂查询拆分为多个子查询：

```
原查询："小鹏汽车 VLA2.0 激光雷达 技术路线"
子查询1："小鹏 VLA2.0 技术架构"
子查询2："小鹏 纯视觉 激光雷达 取舍"
子查询3："VLA2.0 vs 激光雷达 智驾方案对比"
```

## 预算控制

| 预算 | 结果数 | 适用场景 |
|------|--------|---------|
| lite | 5 | 快速事实核查、agent 紧凑上下文 |
| standard | 10 | 默认调研、证据包交接 |
| deep | 20 | 深度研究、用户明确要求更多材料 |

旧命令中的 `minimal / balanced / comprehensive` 仍兼容，分别映射到 `lite / standard / deep`。

## 引擎健康监控

每个引擎有独立的失败计数器：
- 连续失败 3 次 → 熔断 5 分钟
- 成功后重置失败计数
- 熔断期间跳过该引擎
